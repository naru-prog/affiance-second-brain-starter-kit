## Part 1: GitHub アカウントを確認/作成（3分）

### Step 1-1: アカウントの有無を確認

ブラウザで **https://github.com** を開いてください。

| 状況 | 次のステップ |
|------|------------|
| 右上に自分のアイコンが出ている | ✅ ログイン済み → **Step 1-3 へ** |
| 右上に「Sign in」「Sign up」ボタンが出ている | アカウント未作成または未ログイン → **Step 1-2 へ** |

### Step 1-2: アカウントを新規作成（持っていない人だけ）

1. 右上 **「Sign up」** をクリック
2. 順番に入力：
   - **Email**: 普段使うメールアドレス（重要：これがあなたのGitHub IDになります）
   - **Password**: 15文字以上のパスワード（または8文字以上で数字＋小文字混在）
   - **Username**: 半角英数字とハイフンのみ。あとで変更しにくいので慎重に。例: `taro-yamada-2026`
3. 認証用のパズルを解く（人間確認）
4. **「Create account」** クリック
5. 入力したメールアドレスに **6桁のコード** が届くので入力
6. アンケートが出てくるのでテキトーに答えてOK

### Step 1-3: ユーザー名を確認

ログイン後、画面右上のアイコンをクリックすると、自分の **ユーザー名（@で始まる）** が表示されます。

このユーザー名を **メモしておいてください**。後で使います。

例: `@taro-yamada-2026`

**✅ ここまでできたら Part 2 へ**

## Part 2: VS Code から GitHub にサインイン（2分）

ここが**便利ポイント**。VS Code が自動でブラウザを開いて、ワンクリックでサインインできます。

### Step 2-1: my-brain を VS Code で開く

1. **Finder** → 書類 → **my-brain** フォルダを右クリック
2. **「このアプリケーションで開く」** → **「Visual Studio Code」** を選択

または、すでに VS Code が開いている場合は：
1. メニューバーの **File** → **Open Folder...**
2. **my-brain** フォルダを選択 → **「開く」**

### Step 2-2: アカウントメニューを開く

VS Code の **左下の端**を見てください。**人型のアイコン（👤）** があります。それをクリック。

```
   ┌──────────────────┐
   │  📁  📝  🌿  ⊞ │   ← 左サイドバー
   ├──────────────────┤
   │                   │
   │   ファイル一覧    │
   │                   │
   ├──────────────────┤
   │  👤 ← クリック    │   ← 左下の隅
   │  ⚙              │
   └──────────────────┘
```

### Step 2-3: 「GitHub でサインイン」を選択

メニューがポップアップします：

```
   ┌─────────────────────────────────┐
   │ Sign in to Sync Settings        │
   │   ├ Sign in with GitHub  ← これ │
   │   └ Sign in with Microsoft      │
   │                                 │
   │ Backup and Sync Settings...     │
   └─────────────────────────────────┘
```

**「Sign in with GitHub」** をクリック。

### Step 2-4: ブラウザで Authorize

Safari（または既定のブラウザ）が自動で開きます。

1. GitHub のページに **「Authorize Visual-Studio-Code」** という青いボタンが出る
2. クリックして承認
3. 「Open Visual Studio Code」ポップアップが出るので **「開く」** をクリック

### Step 2-5: VS Code に戻って確認

VS Code の左下の **👤アイコン** が、あなたの **GitHubアバター画像** に変わっていれば成功です。

**✅ ここまでできたら Part 3 へ**

## Part 3: my-brain を GitHub に公開（プライベート）（5分）

ここが**核心**。あなたのBrainがクラウドにバックアップされる瞬間です。

### Step 3-1: ソース管理パネルを開く

VS Code の左サイドバーで、**3番目のアイコン（枝分かれしたマーク 🌿）** をクリック。

```
   ┌─┐
   │📁│  ファイル
   ├─┤
   │🔍│  検索
   ├─┤
   │🌿│  ← これ！ソース管理（Source Control）
   ├─┤
   │⊞│  拡張機能
   └─┘
```

### Step 3-2: 「GitHub に公開」ボタンを押す

ソース管理パネルに以下のような表示が出ているはずです：

```
   ┌────────────────────────────────────┐
   │ SOURCE CONTROL                      │
   ├────────────────────────────────────┤
   │  ┌──────────────────────────────┐  │
   │  │  Publish to GitHub          │  │  ← クリック
   │  └──────────────────────────────┘  │
   └────────────────────────────────────┘
```

> 💡 **もし「Initialize Repository」が表示されている場合:** まずそれを押してから、もう一度ソース管理パネルを見ると「Publish to GitHub」が出てきます。

### Step 3-3: リポジトリ名と公開設定を選ぶ

VS Code の上部に **コマンドパレット** が出てきます。

**1. リポジトリ名を入力（または確認）:**

デフォルトで **`my-brain`** と入っているはずです。そのままで OK。

**2. 公開タイプを選ぶ:**

```
   ┌──────────────────────────────────────────┐
   │  Publish to GitHub                       │
   ├──────────────────────────────────────────┤
   │  🔒 Publish to GitHub private repository │  ← ★ 必ずこちら
   │  🌐 Publish to GitHub public repository  │
   └──────────────────────────────────────────┘
```

> ⚠️ **絶対に「private repository」を選んでください。** 「public」を選ぶと、あなたのBrainの中身が **世界中に公開** されます。

### Step 3-4: ファイル選択画面（必要なら）

「どのファイルを含めますか？」と聞かれることがあります。
基本的に **すべてチェック** したまま **「OK」** をクリックして大丈夫です。

> ⚠️ **もし `.env` `secrets/` などのファイルが見えたら、それらは必ずチェックを外してください。** 個人情報や機密情報が含まれている可能性があります（starter-kit では `.gitignore` で自動除外されているはずです）。

### Step 3-5: アップロード完了を待つ

数秒〜数十秒で完了します。

画面右下に **「✓ Successfully published」** のような通知が出れば成功です。

### Step 3-6: GitHub で確認

ブラウザで **https://github.com** を開いて、自分のリポジトリ一覧を確認：

1. 右上のアイコンクリック → **「Your repositories」**
2. 一覧に **`my-brain`** が表示されているはず
3. クリックすると、Brain の中身がすべて入っていることが確認できる
4. リポジトリ名の横に **🔒 Private** と表示されていることを必ず確認

**✅ ここまでできたら Part 4 へ**

## Part 4: 動作テスト（3分）

実際にファイルを変更して、自動同期されるか試してみましょう。

### Step 4-1: テストファイルを編集

1. VS Code で **`CLAUDE.md`** を開く
2. 一番下に1行追加：

   ```
   <!-- テスト: バックアップ動作確認 -->
   ```

3. **`Cmd + S`** で保存

### Step 4-2: ソース管理パネルを確認

左サイドバーの **🌿 ソース管理アイコン** にクリック。
**青い丸の数字（1）** が付いているはずです。これは「変更が1つあります」の意味。

ソース管理パネルを開くと、**`CLAUDE.md`** が「変更されたファイル」として表示されています。

### Step 4-3: コミット（変更を確定）

1. 上部のテキスト入力欄に **コミットメッセージ** を入力：

   ```
   テスト：バックアップ動作確認
   ```

2. その上の **チェックマーク（✓）** ボタンをクリック、または **`Cmd + Enter`**

> 💡 **「No staged changes」と聞かれたら:** 「Yes」をクリック。これは「変更したファイルを全部含めますか？」という確認です。

### Step 4-4: GitHub に同期

コミット後、画面下部のステータスバーに **「Sync Changes」** という青いボタンが出ます。クリック。

または、ソース管理パネルの **「...（三点リーダー）」** メニュー → **「Push」** でもOK。

### Step 4-5: GitHub で確認

ブラウザで GitHub の `my-brain` リポジトリを開いて、**`CLAUDE.md`** をクリック → 一番下に「テスト: バックアップ動作確認」が追加されていれば成功です 🎉

**✅ ここまでできたら Part 5 へ**

## Part 5: 日々のバックアップ習慣（2分）

毎日の作業の終わりに、たった3クリックでバックアップできます。

### 毎日のルーティン（30秒）

1. VS Code の **🌿 ソース管理アイコン** をクリック
2. 上のテキスト欄に「今日やったこと」を1行書く（例: `note記事のアイデアを追加`）
3. **チェックマーク（✓）** をクリック → **「Sync Changes」** をクリック

これだけ。**1日の作業がクラウドに保存**されます。

### より頻繁にバックアップしたい場合

VS Code の設定で **「自動コミット＆プッシュ」** をオンにできます：

1. メニューバー **Code** → **設定** → **設定**
2. 検索欄に `git.autofetch` と入力 → ☑ をオン
3. 検索欄に `git.enableSmartCommit` と入力 → ☑ をオン

これで「変更を保存する → 自動でコミット候補に追加される → ワンクリックで同期」というスムーズな流れになります。

## トラブルシューティング

| 症状 | 原因 | 対処 |
|------|------|------|
| 「Publish to GitHub」ボタンが出ない | Git未初期化 | ソース管理パネルの「Initialize Repository」を先にクリック |
| 「Initialize Repository」も出ない | フォルダが開かれていない | File → Open Folder で my-brain を開く |
| 「Authentication failed」エラー | VS Codeのサインイン期限切れ | Part 2 の手順でもう一度サインイン |
| Sync後もGitHubに反映されない | ブラウザのキャッシュ | GitHub のページを **`Cmd + R`** で再読み込み |
| Public/Private の選択画面が出ない | 既に公開済み | 一度 GitHub で削除してから再試行 |
| 「rejected」エラー | 別のPCで先に変更がある | ソース管理パネル → 「...」 → 「Pull」を先に実行 |

## よくある質問

### Q1. 無料のGitHubアカウントでプライベートリポジトリは何個まで作れますか？

**無制限**です。GitHub Free プランで、プライベートリポジトリは何個でも作れます（2019年から無料化されました）。

### Q2. 容量制限はありますか？

リポジトリ1個あたり **1GBまで推奨**、最大5GBです。テキストベースのBrainなら、何年使っても全く問題ありません（数十MB程度）。

### Q3. 別のMacで復元したい時はどうすれば？

新しいMacで：
1. VS Code をインストール（HOW_TO_INSTALL_VSCODE_MAC.md 参照）
2. VS Code から GitHub にサインイン（Part 2 と同じ手順）
3. **コマンドパレット**（`Cmd + Shift + P`） → `Git: Clone` と入力
4. 自分のリポジトリ一覧から `my-brain` を選択
5. 保存先フォルダを指定 → 完了

これで5分以内に完全復元できます。

### Q4. 「うっかり大事なファイルを消してしまった！」復元できますか？

できます。GitHub のリポジトリページで：
1. 復元したいファイルのあったフォルダを開く
2. 右上の **「History」** をクリック
3. 過去のコミット一覧から、ファイルがあった時点を選択
4. ファイルをクリック → **「Raw」** から内容をコピー → 復元

### Q5. iPhoneやiPadからもBrainを見られますか？

ブラウザで `github.com/<あなたのユーザー名>/my-brain` にアクセスすればファイルを閲覧できます。
本格的な編集には **「Working Copy」**（iOS用Gitアプリ）などの利用がおすすめです。
